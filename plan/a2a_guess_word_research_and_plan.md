# A2A 首字母猜词游戏：需求调研与执行计划

## 1. 调研输入
- 赛事手册：`doc/走向 A2A 互联网 - Second Me 全球首届 A2A 应用开发黑客松 - 开发者手册！.pdf`
- 现有需求：`plan/requirements`

当前已知目标：开发一款**首字母猜词游戏**，支持 **Agent vs Agent**、**Agent vs Human** 对战。

---

## 2. 需求调研结论（从赛事规则反推）

### 2.1 强约束（不满足会影响参赛有效性）
1. **必须是 A2A 场景**：至少两个 AI Agent 之间有自主交互，不能只是单 Agent 工具。
2. **必须基于 SecondMe 能力开发**：手册赛道说明明确要求使用 SecondMe API/平台。
3. **必须提供线上可访问 Demo**：仅提交代码仓库不参与评审。
4. **报名与提交在官方平台完成**：提交入口为 `https://hackathon.second.me`。
5. **组队人数上限 4 人**，可单人参赛。

### 2.2 关键评审导向（决定优先级）
- A2A 场景价值：30%
- 创新度：20%
- 完成度：20%
- 用户选择（真实用户数量与增长）：30%

结论：
- 该项目不仅要“能玩”，还要突出 **A2A 自主博弈** 和 **可传播性**（拉新）。
- 必须尽早接入 OAuth 登录，确保用户统计口径一致（手册说明按 OAuth 授权登录数统计）。

### 2.3 赛道选择建议
本项目最匹配 **赛道二：AI 游戏过大年**。

原因：
- 与“首字母猜词对战”天然契合游戏赛道。
- 游戏场景更容易做围观、排行榜、积分机制，利于用户增长。
- 可快速展示 A2A 高频交互（轮次、出题、抢答、推理）。

## 3. 产品需求定义（面向 MVP）

## 3.1 产品定位
一个支持多模式的 A2A 猜词游戏：
- **模式 A：Agent vs Agent（核心）**
- **模式 B：Human vs Agent（增强可玩性）**
- （可选）**观战模式**：人类观看两位 Agent 实时对战

### 3.2 核心玩法（首字母猜词）
1. 系统给出词语首字母提示（例如 `A _ _ _ _`）。
2. 双方轮流猜词/提问线索（按规则可配置）。
3. 在限定轮次或限定时间内，猜中得分。
4. 多局后按积分决胜，生成对战记录。

### 3.3 MVP 必须功能
1. **SecondMe OAuth 登录**（用于身份与用户数统计）
2. **对战房间系统**（创建/加入/开始/结束）
3. **A2A 自动对战引擎**（两 Agent 自动轮次交互）
4. **人机对战流程**（玩家输入 + Agent 响应）
5. **计分与排行榜**（日榜/总榜至少其一）
6. **结果页分享**（便于传播拉新）

### 3.4 可选加分功能（按时间裁剪）
- Agent 语音播报猜词过程（手册有“语音更有趣”的建议）
- 观战弹幕/评论
- 不同词库主题（春节、科技、影视等）
- 连胜机制、段位机制

## 4. 技术与实现方案

### 4.1 推荐架构
- 前端：Next.js（快速构建页面与部署）
- 后端：Next.js API Routes / Node 服务
- 数据：轻量数据库（Supabase/Postgres/SQLite 按资源选择）
- 身份：SecondMe OAuth2
- Agent 调用：SecondMe API（会话、资料、交互能力）
- 部署：Vercel（优先，便于快速上线 Demo）

### 4.2 模块拆分
1. `auth`：OAuth 登录、会话管理
2. `game-engine`：轮次、规则、胜负判定
3. `agent-orchestrator`：Agent 回合调度与上下文管理
4. `match-service`：房间、对局状态、并发控制
5. `leaderboard`：积分统计与榜单
6. `share`：战报生成与链接分享

## 6. Demo 设计建议（对应评审）

### 6.1 3 分钟演示脚本
1. 10 秒：一句话价值主张（A2A 猜词竞技场）
2. 40 秒：Agent vs Agent 自动开战（展示自主交互）
3. 40 秒：Human vs Agent 实战（展示可玩性）
4. 30 秒：榜单与战报分享（展示增长潜力）
5. 20 秒：技术亮点（SecondMe OAuth + A2A 调度）
6. 40 秒：未来扩展（语音播报、多人锦标赛）

### 6.2 评审映射话术
- A2A 场景价值：强调“不是 AI 工具，而是 AI 与 AI 对战网络”
- 创新度：强调“首字母机制 + Agent 推理策略”
- 完成度：强调“登录-开局-结算-榜单全链路在线可用”
- 用户选择：强调“可分享战报 + 观战入口驱动裂变”

## 7. 风险清单与应对
1. **OAuth 接入延迟**  
   应对：先用本地 mock 页面并行开发，OAuth 一通即切换。
2. **Agent 对战耗时过长**  
   应对：设置单回合超时和默认兜底策略。
3. **线上稳定性不足**  
   应对：减少非核心功能，优先保障“可登录、可开局、可结算”。
4. **用户增长不足影响排名**  
   应对：尽早发布体验链接，强化分享页和邀请文案。
