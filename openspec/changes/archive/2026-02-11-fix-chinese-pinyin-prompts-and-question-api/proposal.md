## Why

当前页面文案和玩法提示偏向“英文单词猜测”，与产品实际目标（通过拼音首字母猜中文常见词语）不一致，导致用户理解偏差。与此同时，题目生成缺少结构化约束，无法稳定输出“拼音首字母 + 正确答案 + 分类提示”的可用题目，影响后续玩法扩展。

## What Changes

- 调整前端游戏提示文案，明确这是“中文词语拼音首字母猜词”玩法（例如 `CF` 可对应“吃饭 / 充分 / 出发”）。
- 新增后端题目生成接口，仅提供服务端 API（暂不接入前端交互）。
- 题目生成结果必须包含：
  - 每个字拼音开头字母序列（2-4 个汉字对应 2-4 个首字母）
  - 正确答案词语
  - 词语分类提示（如动作、动物、水果等）
- 限定题目词语长度为 2-4 个字，并保证拼音首字母与答案逐字对应。

## Capabilities

### New Capabilities

- `chinese-pinyin-game-copy`: 统一页面文案为中文拼音猜词语语义，避免英文单词误导。
- `pinyin-question-generation-api`: 提供生成 2-4 字中文词语题目的后端接口，返回拼音首字母、答案与分类提示。

### Modified Capabilities

- None.

## Impact

- Affected code: 前端游戏页提示文案相关组件（如 `src/app/play/page.tsx` 及其文案依赖）。
- APIs: 新增题目生成接口（建议放在 `src/app/api/...` 路由下），定义统一响应结构供后续前端接入。
- Validation/logic: 新增拼音首字母与答案逐字校验逻辑，确保输出题目满足规则。
- Product: 玩法说明与用户认知统一到“中文拼音猜词”，降低理解成本。
